# Execution Report Template

Use um arquivo por item, no formato `FXXTYY-slug-do-item.execution.md`.

## 1. Resumo do item

- Arquivo de planejamento: `FXXTYY-slug-do-item.md`
- Item de backlog de origem
- Linha correspondente no board: `dev-docs/03-backlog/kanban.md`
- Lingua principal do sistema e da documentacao
- Objetivo do item
- Estado atual
- Proxima tarefa recomendada

## 2. Status das tarefas

Para cada tarefa:
- identificador: `T1`
- status: `pending | in_progress | blocked | done`
- depends on
- resumo curto

## 3. Rodadas de execucao

Para cada rodada:

### Rodada N

- tarefa executada
- modo: `manual | guiado | sequencial`
- objetivo da rodada
- atualizacao aplicada ao `kanban.md`, se houver
- o que foi implementado
- arquivos ou modulos afetados
- validacoes executadas
- atualizacao de `AGENTS.md` e `README.md`, se aplicavel
- checks de seguranca e privacidade
- falhas ou pendencias
- ajustes no plano
- licoes aprendidas
- decisao final da rodada

## 4. Bloqueios e pendencias

- bloqueio
- impacto
- acao sugerida

## 5. Proximos passos

- proxima tarefa segura
- pre-requisitos
- validacoes esperadas
- mensagens de commit sugeridas no mesmo idioma principal da documentacao e dos artefatos
