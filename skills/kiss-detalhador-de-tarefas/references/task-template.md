# Task Template

## 1. Resumo do item

- Arquivo sugerido no formato `FXXTYY-slug-do-item.md`
- Item de backlog de origem
- Linha correspondente no board: `dev-docs/03-backlog/kanban.md`
- Lingua principal do sistema e da documentacao
- Objetivo
- Criterio de aceite
- Dependencias conhecidas
- Impacto de seguranca
- Impacto de privacidade

## 2. Impacto tecnico

- Sistemas ou modulos afetados
- APIs ou contratos afetados
- Dados ou schema afetados
- Riscos relevantes

## 3. Tarefas

Para cada tarefa:
- identificador, como `T1`
- titulo
- objetivo
- depends on
- validation
- done when
- owner sugerido ou trilha

## 4. Subtarefas ou checklist tecnico

- `T1.1` passo 1
- `T1.2` passo 2
- `T1.3` passo 3

## 5. Testes e verificacoes

- teste esperado
- validacao de seguranca ou permissao
- validacao de privacidade ou tratamento de dados
- observabilidade ou logs
- verificacao manual, se houver

## 6. Definition of Done

- criterio 1
- criterio 2
- criterio 3

## 7. Sequencia recomendada

- pre-requisito
- execucao principal
- integracao
- validacao final

## 8. Atualizacao operacional

- se o item estiver pronto para execucao, atualizar `dev-docs/03-backlog/kanban.md` para `ready`
- registrar proxima acao esperada no board
