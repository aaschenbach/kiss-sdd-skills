# PRD Template

Use este modelo como estrutura base. Remova secoes irrelevantes e marque lacunas explicitamente.

## 1. Resumo executivo

- Nome provisiorio do produto ou iniciativa
- Nomes alternativos considerados, se relevante
- Lingua principal do sistema
- Lingua principal da documentacao
- Objetivo de negocio
- Problema que sera resolvido
- Publico-alvo prioritario
- Resultado esperado do MVP

## 2. Contexto

- Situacao atual
- Oportunidade identificada
- Evidencias disponiveis
- Restricoes conhecidas

## 3. Problema

- Declaracao do problema
- Impacto no usuario
- Impacto no negocio
- Alternativas atuais e suas limitacoes

## 4. Publico-alvo

- Usuario principal
- Comprador ou decisor, se diferente
- Operador ou administrador, se diferente
- Segmento inicial priorizado

## 5. Proposta de valor

- Promessa central do produto
- Principais beneficios
- Diferenciais esperados

## 6. Naming

- Nome recomendado
- Racional do nome
- Alternativas consideradas
- Risco aparente de colisao de mercado
- Pendencias de validacao juridica, se houver

## 7. Objetivos e metricas

- Metricas de resultado
- Metricas de uso ou ativacao
- Metricas de guarda
- Como medir

## 8. Escopo do MVP

### Dentro do escopo

- Funcionalidade 1
- Funcionalidade 2
- Funcionalidade 3

### Fora de escopo

- Item adiado 1
- Item adiado 2

## 9. Jornada e fluxos principais

Descreva os principais fluxos ponta a ponta.

Para cada fluxo, explicite:
- gatilho
- passos
- excecoes
- resultado esperado

## 10. Requisitos funcionais

Enumere requisitos em formato verificavel.

Exemplo:
- RF-01: O sistema deve permitir ...
- RF-02: O usuario deve conseguir ...

## 11. Requisitos nao funcionais

Cubra conforme aplicavel:
- desempenho
- seguranca
- privacidade
- auditoria
- disponibilidade
- observabilidade
- acessibilidade

## 12. Seguranca e privacidade

- Tipos de dados envolvidos
- Dados pessoais ou dados sensiveis, se houver
- Perfis e regras de acesso
- Requisitos de consentimento, retencao, exclusao ou auditoria
- Riscos principais e mitigacoes esperadas no nivel de produto
- Pendencias para avaliacao juridica, de seguranca ou compliance

## 13. Dados e instrumentacao

- Entidades principais
- Dados minimos necessarios
- Eventos a instrumentar
- Painel ou relatorio desejado

## 14. Definition of Done e qualidade

Defina o minimo necessario para considerar a entrega pronta com seguranca.

Inclua conforme aplicavel:
- fluxos principais funcionando de ponta a ponta
- criterios de aceite validados
- tratamento de erros criticos
- instrumentacao minima habilitada
- testes essenciais definidos e executados
- validacao de permissoes e acesso quando houver dados sensiveis
- observabilidade, auditoria ou rollback quando relevante

Nao detalhe aqui todos os casos de teste. O objetivo e registrar o padrao minimo de qualidade da entrega.

## 15. Dependencias

- Integracoes
- Times envolvidos
- Fornecedores ou terceiros
- Premissas tecnicas

## 16. Riscos

- Risco
- Impacto
- Mitigacao

## 17. Questoes em aberto

- Questao
- Dono
- Proximo passo

## 18. Criterios de aceite

Defina criterios claros para considerar o MVP pronto para lancamento interno ou externo.

## 19. Expansoes opcionais apos o PRD

Use esta secao apenas quando ajudar o handoff.

- Skill opcional recomendada apos o PRD, se aplicavel
- Quando faz sentido acionar essa skill
- O que ela deve aprofundar
- O que permanece fora do escopo do PRD principal
